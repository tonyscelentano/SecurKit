SecurKit TUI — screen tour

SVG screenshots captured via Textual's app.save_screenshot(). These are pixel-faithful renders of what you'll see in a real terminal.

1. Welcome 2. Encrypt (empty) 3. Encrypt (after Suggest) 4. Encrypt (weak + mismatch) 5. Decrypt 6. Hotlines list 7. Hotlines SEC 8. About

1. Welcome screen (first run)

Shown once. The "lost passphrase = lost data" warning is the most important thing on this screen — every consumer encryption tool needs to scream this.

2. Encrypt pane (empty)

Drops here after dismissing the welcome. Plain-English instruction at the top; source folder, output bundle, passphrase fields, Suggest button.

3. Encrypt pane (after pressing Suggest)

Suggest fills both passphrase fields with a 7-word diceware passphrase (~56 bits). Confirm shows "✓ matches". Strength meter goes green. Feedback tells the user to write the passphrase down before continuing.

4. Encrypt pane (weak passphrase + mismatch)

User typed "password123" and a different value into confirm. Strength meter is red, zxcvbn flags the common password, confirm field shows "✗ doesn't match" in red. But — per the "coach but don't block" stance — the Encrypt button isn't disabled.

5. Decrypt pane

Mirror of Encrypt. Single passphrase field (no confirm — the bundle's tag is the source of truth). Helper text at the top reminds the user that nothing is written to disk if the bundle fails verification.

6. Hotlines pane (list)

All 15 US-focused intake channels. Each entry shows name + agency. Master/detail layout — click an entry to see the details.

7. Hotlines pane (SEC selected)

Detail pane shows scope, contact channels (web/phone), retaliation protection statute, notes. This is the "portfolio differentiator" — most encryption tools stop at the bundle.

8. About

Version, stack summary, the "talk to a lawyer first" reminder, threat-model pointer.
